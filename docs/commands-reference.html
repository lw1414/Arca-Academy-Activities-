[01]Servers (long-running)

# Static site server
node serve.mjs                                  # → http://localhost:3000

# Express server (POST /api/contact → Google Sheets)
node server.js                                  # → http://localhost:3000

# Briefing daemon (cron + Run-now API on :3100)
node morning-briefing.mjs watch

[02]Morning briefing

Preview (no email sent)

node morning-briefing.mjs preview               # real Supabase leads (24h)
node morning-briefing.mjs preview --demo        # 6 sample leads

Send for real

node morning-briefing.mjs run                   # real leads, via Resend
node morning-briefing.mjs run --demo            # demo leads, via Resend
node morning-briefing.mjs run --gmail           # real leads, via Gmail
node morning-briefing.mjs run --demo --gmail    # demo leads, via Gmail

Dry run (renders, prints stats, no email, no DB log)

node morning-briefing.mjs run --demo --dry
node morning-briefing.mjs run --gmail --dry

[04]Playwright tests

npx playwright test                             # all tests, headless
npx playwright test --headed                    # all tests, watch browser
npx playwright test --ui                        # interactive UI
npx playwright test --debug                     # step-through debugger
npx playwright show-report                      # open HTML report

# Specific spec files
npx playwright test tests/form.spec.mjs         # 20 form submissions
npx playwright test tests/links.spec.mjs        # link integrity audit
npx playwright test tests/briefing.spec.mjs     # briefing preview smoke

# Single test by line number
npx playwright test tests/form.spec.mjs:14

[11]Process control

# Check what is listening on each port
netstat -ano | findstr :3000
netstat -ano | findstr :3100

# Kill all node processes
Get-Process node | Stop-Process

# Kill a specific PID (from netstat)
Stop-Process -Id 12345

[12]Troubleshooting

SymptomFix
"Daemon not running on :3100" in dashboardRun node morning-briefing.mjs watch
"RESEND_API_KEY missing"Open .env, set RESEND_API_KEY=re_...
"GMAIL_USER / GMAIL_PASS_ARCA missing"Open .env, fill both
"You can only send testing emails…" (Resend)Change BRIEFING_TO to your Resend-registered email, OR verify a domain
Dashboard 401 / RLS errorRun supabase/dashboard-read-policy.sql
"relation … does not exist"Run supabase/dashboard-tables.sql
Briefings not in Email Log tabRun supabase/email-log-add-briefing-source.sql
Port 3000 / 3100 already in useGet-Process node | Stop-Process then restart
Pro tip. Start two PowerShell terminals at the project root and leave them running: node serve.mjs in one, node morning-briefing.mjs watch in the other. The dashboard will detect both daemons automatically.

Release notes.

Everything that shipped, dated and signed. We push small, often, and in public.

v2.4.0May 28, 2026

Adaptive flake-quarantine

  • Tests flagged as flaky on three consecutive runs are auto-quarantined and routed to a triage queue.
  • New quarantine budget setting per project — alerts when the budget is exceeded.
  • Slack/Teams notifications now include the suspected diff range.
v2.3.1May 14, 2026

Faster trace upload on Windows runners

  • Trace compression now happens in-process instead of via subprocess — 38% faster on average for Windows self-hosted runners.
  • Fixed a regression where retried tests were missing video on certain browser configurations.
v2.3.0April 30, 2026

Visual diff thresholds

  • Per-test perceptual diff thresholds (defaults to 0.2).
  • New "ignore region" syntax for masking dynamic UI.
  • Beta: AI-assisted baseline regeneration with one-click approve.
v2.2.0April 9, 2026

Multi-workspace billing

  • Agencies and consultancies can now manage many client workspaces under one billing entity.
  • Per-workspace seat allocation and usage caps.
v2.1.0March 18, 2026

Native GitHub Checks integration

  • Tester.io now posts a single Check per PR with sub-runs per browser/viewport.
  • Required-checks now fail fast on the first regression.
v2.0.0February 4, 2026

The grand re-shape

  • Rewrote the runner core in Rust for a 2.4× throughput improvement on parallel suites.
  • New dashboard with cohort-level flake analytics.
  • Migration guide and codemod for v1 suites available.
